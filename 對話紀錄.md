## 二、HW3-1：Naive DQN 與 Experience Replay

**使用者：**  
基本版 DQN 要怎麼設計？

**AI Agent：**  
我會建立三個主要部分：

1. `ReplayBuffer`：儲存 `(state, action, reward, next_state, done)`。
2. `QNetwork`：輸入 64 維 state，輸出 4 個 action 的 Q-value。
3. 訓練迴圈：使用 epsilon-greedy 探索，並用 Bellman target 更新 Q-network。

**使用者：**  
Replay Buffer 在這裡的作用是什麼？

**AI Agent：**  
如果 agent 每走一步就直接拿當下 transition 更新模型，前後資料高度相關，容易讓神經網路不穩定。Replay Buffer 會把過去經驗存起來，訓練時隨機抽 mini-batch，讓資料分布比較接近 i.i.d.，也能重複利用舊經驗。

**使用者：**  
那 state 要怎麼處理？

**AI Agent：**  
`GridBoard.render_np()` 會輸出 shape 類似 `(4, 4, 4)` 的 board representation。程式中用 `preprocess_state()` 把它 flatten 成 64 維，再轉成 `torch.FloatTensor`：

```python
def preprocess_state(state):
    state_flat = state.flatten()
    return torch.FloatTensor(state_flat).unsqueeze(0)
```

**使用者：**  
訓練目標怎麼算？

**AI Agent：**  
Naive DQN 使用標準 Bellman target：

```text
target = reward + gamma * max_a Q(next_state, a)
```

在程式中對應：

```python
target_qval = r_batch + gamma * max_nqval * (~done_batch)
```

如果 episode 結束，`done=True`，就不再加上 future reward。

**使用者：**  
先讓 agent 在固定地圖上學習就好。

**AI Agent：**  
已在 `hw3_1_naive.py` 中使用 `Gridworld(size=4, mode='static')`。這代表 Goal、Pit、Wall、Player 的初始位置固定，agent 可以先學習穩定的路徑。訓練完成後會輸出 loss curve 和 reward moving average 到：

```text

```

## 三、HW3-2：Double DQN 與 Dueling DQN

**使用者：**  
基本 DQN 完成後，下一題要比較 Double DQN 和 Dueling DQN。

**AI Agent：**  
我會在 `hw3_2_variants.py` 裡加入兩種變體，並把環境改成 `player` mode。這個模式會固定 Goal、Pit、Wall，但 Player 的起點隨機，比 `static` 更能測試泛化能力。

**使用者：**  
Double DQN 解決什麼問題？

**AI Agent：**  
Naive DQN 用同一個 network 選 action 和估計 target value，容易高估 Q-value。Double DQN 把這兩件事拆開：

- online network：選擇下一步最佳 action
- target network：估計該 action 的 Q-value

程式中對應：

```python
next_action = online_net(ns_batch).argmax(1).unsqueeze(1)
target_qval = r_batch + gamma * target_net(ns_batch).gather(1, next_action).squeeze(1) * (~done_batch)
```

**使用者：**  
Dueling DQN 又是改哪裡？

**AI Agent：**  
Dueling DQN 是改網路架構。它把 Q-value 拆成：

```text
Q(s, a) = V(s) + A(s, a) - mean(A(s, a))
```

其中：

- `V(s)`：這個 state 本身有多好
- `A(s,a)`：某個 action 相對於其他 action 有多好

這在很多 action 差異不大的 state 裡很有幫助，因為模型可以更快學到 state value。

**使用者：**  
兩個模型要怎麼比較？

**AI Agent：**  
我會讓 Double DQN 和 Dueling DQN 都訓練 2000 episodes，使用 reward moving average 比較表現。輸出圖檔為：

```text
plots/hw3_2_comparison.png
```

## 四、HW3-3：改寫成 PyTorch Lightning

**使用者：**  
下一題要把 DQN 改成 PyTorch Lightning。

**AI Agent：**  
我會把模型封裝成 `DQNLightning(L.LightningModule)`，把 online network、target network、optimizer、scheduler 放進 LightningModule。不過 RL 的資料不是固定 dataset，而是 agent 跟環境互動產生，所以訓練 loop 仍保留客製化形式。

**使用者：**  
這樣還算 Lightning 嗎？

**AI Agent：**  
程式中已經把模型、optimizer 設定和 scheduler 放進 LightningModule：

```python
class DQNLightning(L.LightningModule):
    ...
    def configure_optimizers(self):
        optimizer = torch.optim.Adam(self.parameters(), lr=self.hparams.lr)
        scheduler = CosineAnnealingLR(optimizer, T_max=1000)
        return [optimizer], [scheduler]
```

因為 RL loop 需要持續和 `Gridworld` 互動，所以目前採取「LightningModule 管理模型與訓練設定，外部 loop 負責環境互動」的折衷設計。

**使用者：**  
random mode 比較難，有沒有加穩定訓練的方法？

**AI Agent：**  
有，`hw3_3_lightning.py` 中加入三個穩定化技巧：

1. Gradient clipping：避免 gradient explosion。
2. CosineAnnealingLR：讓 learning rate 慢慢下降。
3. Soft target update：用 Polyak averaging 平滑更新 target network。

Soft update 程式如下：

```python
target_param.data.copy_(self.tau * online_param.data + (1.0 - self.tau) * target_param.data)
```

這一題的輸出是？

**AI Agent：**  
這題使用 `Gridworld(size=4, mode='random')`，所有物件位置都隨機。訓練完會輸出：

## 五、HW3-4 Bonus：Rainbow DQN

**使用者：**  
bonus 想做 Rainbow DQN，因為 random mode 太難了。

**AI Agent：**  
Rainbow DQN 是適合這個場景的方向。Random mode 中 Player、Goal、Pit、Wall 全部隨機，agent 不能只記固定路線，而要學會「靠近 Goal、避開 Pit、繞過 Wall」這類抽象空間關係。

**使用者：**  
Rainbow 要包含哪些 component？

**AI Agent：**  
完整 Rainbow 通常包含多個技巧。這份程式在 `hw3_4_rainbow.py` 中實作了五個實用 component：

1. Double DQN
2. Dueling Network
3. Prioritized Experience Replay
4. Multi-step Return
5. Noisy Nets

其中 Distributional RL / C51 被刻意省略，因為 4x4 Gridworld 的 reward structure 很簡單，加入 C51 會大幅增加 projection 和 categorical loss 的複雜度，但收益有限。

**使用者：**  
NoisyNet 是取代 epsilon-greedy 嗎？

**AI Agent：**  
對。`NoisyLinear` 會在權重和 bias 上加入可學習的 noise：

```text
w = mu_w + sigma_w * epsilon_w
```

這讓 network 可以自己學習什麼時候該探索，而不是人工設定 epsilon decay。

**使用者：**  
Prioritized Replay 怎麼做？

**AI Agent：**  
程式使用 `SumTree` 實作 PER。TD-error 越大的 transition priority 越高，被抽到的機率越大。為了修正 sampling bias，loss 會乘上 importance-sampling weight：

```python
loss = (w_b * F.mse_loss(q_pred, q_target, reduction='none')).mean()
```

訓練後再用新的 TD-error 更新 priority：

```python
per_buffer.update_priorities(indices, td_errors)
```

**使用者：**  
Multi-step Return 的用途是什麼？

**AI Agent：**  
一般 DQN 只看一步 reward，credit assignment 比較慢。Multi-step return 會把連續 n 步 reward 壓成一個 transition：

```text
R_t = r_t + gamma r_{t+1} + gamma^2 r_{t+2}
```

在程式中 `n_step=3`，可以讓 agent 更快接收到長期回饋訊號。

**使用者：**  
最後 Rainbow 的訓練設定是？

**AI Agent：**  
`train_rainbow()` 的主要設定如下：

- episodes：3000
- hidden_dim：128
- learning rate：5e-4
- gamma：0.99
- batch size：64
- n-step：3
- target update：每 200 global steps hard update
- replay buffer capacity：10000
- PER alpha：0.6
- PER beta：從 0.4 anneal 到 1.0
- NoisyNet sigma_init：0.5
- max episode steps：60
